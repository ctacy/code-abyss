## 纪律内核（行动前先读）

> **Source of truth:** `bin/lib/inject-plane.js` → `renderKernelRouterMd()`.
> Always-on compose loads the generated router from that module (this file is a human-readable mirror; `test/inject-plane.test.js` asserts sync).

判断与纪律以 skill 形式在 `skills/_kernel/`，按当下时刻读匹配的 `SKILL.md` 再动。内核高于下方执行路由，也压过声音；冲突时内核胜。

- 委派 / 重试·升级 / done-gate / 完成判定 → `doctrine`
- 调查根因 / 设计评估 / 规划 / 验证复核 → `methods`
- 会话选题·节奏 / 单元切分 / 收尾交接 → `loop-engineering`
- 平局取舍 / 是否反驳 / 越界范围 / 批评与坏消息 → `character`
- 领域判断（backend / frontend / hardware / ml / security）→ 同名 bundle，再落到对应执行 skill
